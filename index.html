Broken Access Control in Real Applications

OWASP Mapping: A01 – Broken Access Control, A04 – Insecure Design

This project focuses on identifying authorization failures that occur after authentication, including IDOR, horizontal privilege escalation, and vertical privilege escalation.

What I Did

  • Mapped role-based access assumptions against backend enforcement
  • Tested cross-user and cross-role resource access
  • Validated whether authorization checks were enforced server-side

Key Security Insight

Authentication does not equal authorization. Most critical breaches happen when systems trust authenticated users too much.

Why a CTO Should Care

Broken access control is the leading cause of large-scale data exposure. A single missed authorization check can expose customer data, violate compliance requirements, and permanently damage trust.

Cloud IAM Privilege Escalation Path Analysis

OWASP Mapping: A05 – Security Misconfiguration, A04 – Insecure Design

This project analyzes cloud IAM policies to identify indirect privilege escalation paths created by permission combinations and misconfigured trust relationships.

What I Did

  • Analyzed IAM policies as permission graphs rather than isolated rules
  • Mapped escalation paths involving PassRole, AssumeRole, and policy attachment
  • Identified full-account compromise paths without exploiting software flaws

Key Security Insight

Cloud breaches rarely require exploits. They succeed by abusing configuration and design mistakes in identity systems.

Why a CTO Should Care

IAM failures can lead to total cloud account compromise. These incidents are costly, difficult to detect early, and often result in complete loss of control over production environments.

Credential Abuse & Authentication Failure Detection

OWASP Mapping: A07 – Identification and Authentication Failures, A09 – Security Logging and Monitoring Failures

This project simulates real-world authentication traffic to detect brute-force and credential-stuffing attacks that bypass basic rate limiting and lockout controls.

What I Did

  • Correlated failed and successful login attempts over rolling time windows
  • Identified behavioral patterns instead of relying on static thresholds
  • Focused on detection gaps rather than password complexity rules

Key Security Insight

Credential abuse succeeds not because passwords are weak, but because detection logic is shallow or nonexistent.

Why a CTO Should Care

Credential stuffing leads directly to account takeovers, fraud, and customer trust loss. Early detection significantly reduces incident scope and response costs.
